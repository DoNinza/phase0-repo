## 원본과 다른 점 (의도적 변경, 수치 아님)

- `t_annual()`은 이제 하드코딩된 4개 시나리오만 받지만, 원본과 완전히 동일한
  출력을 낸다 — 구조만 재편.
- **미세 발견 1건**: 원본 `t_cost_sens`는 Conservative 비용을 `.0047`로
  하드코딩했는데, 이는 `Base(0.36%) × 1.3 = 0.468%`를 표시 단계에서 반올림한
  값(0.47%)을 계산에 재사용한 것이다. 반올림 없이 그대로 계산하면 `E_trade =
  -0.048%`이며, 원본 표기(`-0.050%`)와 0.002%p 차이가 난다. 판정 방향에는
  영향 없음(둘 다 음수 → Conservative 미통과는 동일)이지만, 이 자체가
  기획안이 R-12에서 지적한 "표시값을 다시 계산에 흘려 넣는" 패턴의 축소판이라
  일부러 남겨두고 테스트로 고정했다(`tests/test_tables.py`).

## Phase 0 Blocker 해소 (§5.6) — 2026-07-14

`costs.yaml`의 `외부 확인 필요` 항목 2건이 모두 실측값으로 대체되어 Blocker가
해소됐다. Base 왕복비용은 **0.36%(문서 원본 가정치) → 0.3854%(실측)** 로 갱신:

- `brokerage_commission_roundtrip` — 사용자 실계좌(한국투자증권 뱅키스 온라인매매)
  확인 결과 0.0140527%(편도, KRX). 왕복 2배 적용 = 0.0281054%. 기존 0.01%
  임시값 대비 약 2.8배 상향.
- `exchange_fees` — KIS 공식 수수료안내 페이지 기준 유관기관제비용율
  0.00363960%(편도, 2025-10-27 고시). 왕복 추정 0.0073%로 Base에 반영.

이 갱신에 따라 `t_r01_verify`처럼 하드코딩된 예시(원표 조건 C=0.3% 고정)를
제외한 모든 표·테스트가 새 Base(0.3854%) 기준으로 자동 재계산되며,
`tests/test_costs.py`·`tests/test_tables.py`의 관련 assertion도 이 커밋에서
함께 갱신했다. 앞으로 비용이 또 바뀌면 `costs.yaml`의 `value`만 고치고
`pytest`를 돌려 실패하는 assertion만 새 값으로 맞추면 된다.

## Project Blocker 해소 (§5.6) — 2026-07-14

KIS 약관상 자동매매(Open API 기반 개인 알고리즘 거래) 금지 조항 여부를 사용자가
직접 확인 — **금지 조항 없음**. 조사 과정에서 함께 확인된 정황: KRX 알고리즘거래
사전신고 의무는 개인이 아닌 회원(증권사) 대상이며 파생상품계좌 한정 조항이라
이 프로젝트(KOSPI 주식 데이트레이딩)와는 무관, "프로그램매매" 규제(사이드카 등)는
동시 5억원 이상 대량 주문의 기관·외국인 대상이라 개인 알고리즘 거래는 해당 없음,
업계 전반(KIS 2022~, 토스증권 2026~)이 개인용 오픈API 자동매매를 활발히
마케팅하는 정착된 상품 카테고리. 단, 이용약관 원문 전체를 읽고 확정한 법률
자문은 아니며 사용자 본인 확인에 근거한다.

## 첫 전략 VCB-Gap + G0 실데이터 백테스트 (STAGE 7 항목 9·10) — 2026-07-15

Phase 0 문서는 계산 엔진·Gate 구조만 다루고 전략 자체는 정의하지 않았다.
사용자가 "다른 사람 방식(딥러닝+100여 전략)을 바로 차용하지 말고, 규칙
기반 전략 하나부터 G0를 통과시켜보자"고 결정 → fable5(claude-fable-5)에게
설계를 위임해 받은 첫 전략을 실제 10년치 KRX 데이터로 검증했다.

**전략(`phase0/strategy/vcb_gap.py`, "VCB-Gap")**: 모멘텀 지속형. 전일이
20일 신고가를 거래량 1.5배 이상 동반해 돌파하며 강하게(CLV≥0.70) 마감했고,
오늘 시가 갭이 −0.5%~+2.0%(모멘텀을 부정하지도, 소진하지도 않은 범위)이며
ATR%가 1.0~5.0%(너무 조용하지도 과열되지도 않음)일 때 시가에 진입. 목표가/
손절가는 ATR% 배수(k_target/k_stop). D-1까지의 일봉 + 오늘 시가만 사용해
분봉 이력 없이도(위 §분봉 이력 범위 참고) 백테스트 가능하도록 설계됨.

**실행 결과** (`scripts/run_g0_backtest.py`, DEFAULT_CANDIDATES 20종목,
2016-07~2026-07 실KRX 데이터, 사전 등록 격자 6개 조합 전부):

|---|---|---|---|---|---|
| 1.2 | 0.75 | 685 | −0.638% | −0.633% | insufficient_sample |
| 1.2 | 1.00 | 685 | −0.608% | −0.613% | insufficient_sample |
| 1.5 | 0.75 | 685 | −0.619% | −0.619% | insufficient_sample |
| 1.5 | 1.00 | 685 | −0.590% | −0.597% | insufficient_sample |
| 2.0 | 0.75 | 685 | −0.602% | −0.598% | insufficient_sample |

**정직한 결론**: 6개 조합 전부 낙관·보수 경로 모두 마이너스로 일관됨.
표본(685건)이 G0 기준(1,000건)에 못 미쳐 형식적 판정은 "표본 부족"이지만,
부호가 모든 조합·양쪽 경로에서 한결같이 음수라 표본을 더 채운다고 뒤집힐
가능성은 낮아 보인다 — **이 진입 규칙 자체가 KOSPI 대형주·2016~2026
구간에서 비용을 넘는 엣지를 보여주지 못했다**는 것이 현재까지 가장 정직한
해석이다. fable5가 설계 시 명시한 "장세 의존성으로 인한 기각은 버그가
아니라 정당한 G0 기각"이 실제로 발생한 사례로 본다. 목표/손절 배수를
격자 밖에서 추가로 찾는 것은 데이터마이닝이므로 하지 않았다.

**다음으로 고려할 것(미결정, 사용자 판단 대기)**: (a) 진입 로직(C1~C6) 자체를
다른 가설로 다시 설계, (b) 기간을 나눠 장세별(추세장/박스권) 부분 표본을
따로 봐서 regime-dependence 확인, (c) 후보 종목을 늘려 표본 1,000건을
채운 뒤 재확인(부호가 이미 일관되게 음수라 우선순위는 낮음).

## 두 번째 전략 GDR(갭하락 반등) — 2026-07-15

사용자가 "아예 다른 가설로 가보자"고 결정 → fable5에게 VCB-Gap의 실패
진단(당일청산 제약이 '지속형' 가설과 구조적으로 안 맞았을 가능성)을
그대로 전달하고 재설계 요청. fable5가 제시한 논거: VCB-Gap의 실측
E_trade(−0.58~−0.64%)가 '엣지 없음' 중립 기준선(−0.12%)보다도 나빴다는
것은 이 유니버스에 단기 반전(reversal) 성향이 실재한다는 방증이므로,
정확히 반대 부호(과매도 이탈 매수)를 조건화하면 그 반전을 포착할 수
있다는 것.

**전략(`phase0/strategy/gap_rebound.py`, "GDR — Gap-Down Rebound")**:
평균회귀형. 하락추세가 아닌 종목(종가≥0.98×SMA20)이 비정상적으로
갭하락(−1.2%~−4.5%, 종목 자체 ATR% 대비로도 비정상)했지만 전일 자체가
이미 급락(−5% 초과)한 건 아닐 때 시가에 매수, 목표가는 전일종가 방향
갭의 부분 되메움(f_fill), 손절은 ATR% 기반. 12월 배당락 구간(12/22~31)은
기계적 갭이라 제외.
