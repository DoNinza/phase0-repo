## 원본과 다른 점 (의도적 변경, 수치 아님)

- `t_annual()`은 이제 하드코딩된 4개 시나리오만 받지만, 원본과 완전히 동일한
  출력을 낸다 — 구조만 재편.
- **미세 발견 1건**: 원본 `t_cost_sens`는 Conservative 비용을 `.0047`로
  하드코딩했는데, 이는 `Base(0.36%) × 1.3 = 0.468%`를 표시 단계에서 반올림한
  값(0.47%)을 계산에 재사용한 것이다. 반올림 없이 그대로 계산하면 `E_trade =
  -0.048%`이며, 원본 표기(`-0.050%`)와 0.002%p 차이가 난다. 판정 방향에는
  영향 없음(둘 다 음수 → Conservative 미통과는 동일)이지만, 이 자체가
  기획안이 R-12에서 지적한 "표시값을 다시 계산에 흘려 넣는" 패턴의 축소판이라
  일부러 남겨두고 테스트로 고정했다(`tests/test_tables.py`).

## Phase 0 Blocker 해소 (§5.6) — 2026-07-14

`costs.yaml`의 `외부 확인 필요` 항목 2건이 모두 실측값으로 대체되어 Blocker가
해소됐다. Base 왕복비용은 **0.36%(문서 원본 가정치) → 0.3854%(실측)** 로 갱신:

- `brokerage_commission_roundtrip` — 사용자 실계좌(한국투자증권 뱅키스 온라인매매)
  확인 결과 0.0140527%(편도, KRX). 왕복 2배 적용 = 0.0281054%. 기존 0.01%
  임시값 대비 약 2.8배 상향.
- `exchange_fees` — KIS 공식 수수료안내 페이지 기준 유관기관제비용율
  0.00363960%(편도, 2025-10-27 고시). 왕복 추정 0.0073%로 Base에 반영.

이 갱신에 따라 `t_r01_verify`처럼 하드코딩된 예시(원표 조건 C=0.3% 고정)를
제외한 모든 표·테스트가 새 Base(0.3854%) 기준으로 자동 재계산되며,
`tests/test_costs.py`·`tests/test_tables.py`의 관련 assertion도 이 커밋에서
함께 갱신했다. 앞으로 비용이 또 바뀌면 `costs.yaml`의 `value`만 고치고
`pytest`를 돌려 실패하는 assertion만 새 값으로 맞추면 된다.

## Project Blocker 해소 (§5.6) — 2026-07-14

KIS 약관상 자동매매(Open API 기반 개인 알고리즘 거래) 금지 조항 여부를 사용자가
직접 확인 — **금지 조항 없음**. 조사 과정에서 함께 확인된 정황: KRX 알고리즘거래
사전신고 의무는 개인이 아닌 회원(증권사) 대상이며 파생상품계좌 한정 조항이라
이 프로젝트(KOSPI 주식 데이트레이딩)와는 무관, "프로그램매매" 규제(사이드카 등)는
동시 5억원 이상 대량 주문의 기관·외국인 대상이라 개인 알고리즘 거래는 해당 없음,
업계 전반(KIS 2022~, 토스증권 2026~)이 개인용 오픈API 자동매매를 활발히
마케팅하는 정착된 상품 카테고리. 단, 이용약관 원문 전체를 읽고 확정한 법률
자문은 아니며 사용자 본인 확인에 근거한다.

## 첫 전략 VCB-Gap + G0 실데이터 백테스트 (STAGE 7 항목 9·10) — 2026-07-15

Phase 0 문서는 계산 엔진·Gate 구조만 다루고 전략 자체는 정의하지 않았다.
사용자가 "다른 사람 방식(딥러닝+100여 전략)을 바로 차용하지 말고, 규칙
기반 전략 하나부터 G0를 통과시켜보자"고 결정 → fable5(claude-fable-5)에게
설계를 위임해 받은 첫 전략을 실제 10년치 KRX 데이터로 검증했다.

**전략(`phase0/strategy/vcb_gap.py`, "VCB-Gap")**: 모멘텀 지속형. 전일이
20일 신고가를 거래량 1.5배 이상 동반해 돌파하며 강하게(CLV≥0.70) 마감했고,
오늘 시가 갭이 −0.5%~+2.0%(모멘텀을 부정하지도, 소진하지도 않은 범위)이며
ATR%가 1.0~5.0%(너무 조용하지도 과열되지도 않음)일 때 시가에 진입. 목표가/
손절가는 ATR% 배수(k_target/k_stop). D-1까지의 일봉 + 오늘 시가만 사용해
분봉 이력 없이도(위 §분봉 이력 범위 참고) 백테스트 가능하도록 설계됨.

**실행 결과** (`scripts/run_g0_backtest.py`, DEFAULT_CANDIDATES 20종목,
2016-07~2026-07 실KRX 데이터, 사전 등록 격자 6개 조합 전부):

|---|---|---|---|---|---|
| 1.2 | 0.75 | 685 | −0.638% | −0.633% | insufficient_sample |
| 1.2 | 1.00 | 685 | −0.608% | −0.613% | insufficient_sample |
| 1.5 | 0.75 | 685 | −0.619% | −0.619% | insufficient_sample |
| 1.5 | 1.00 | 685 | −0.590% | −0.597% | insufficient_sample |
| 2.0 | 0.75 | 685 | −0.602% | −0.598% | insufficient_sample |
| 2.0 | 1.00 | 685 | −0.577% | −0.577% | insufficient_sample |

**정직한 결론**: 6개 조합 전부 낙관·보수 경로 모두 마이너스로 일관됨.
표본(685건)이 G0 기준(1,000건)에 못 미쳐 형식적 판정은 "표본 부족"이지만,
부호가 모든 조합·양쪽 경로에서 한결같이 음수라 표본을 더 채운다고 뒤집힐
가능성은 낮아 보인다 — **이 진입 규칙 자체가 KOSPI 대형주·2016~2026
구간에서 비용을 넘는 엣지를 보여주지 못했다**는 것이 현재까지 가장 정직한
해석이다. fable5가 설계 시 명시한 "장세 의존성으로 인한 기각은 버그가
아니라 정당한 G0 기각"이 실제로 발생한 사례로 본다. 목표/손절 배수를
격자 밖에서 추가로 찾는 것은 데이터마이닝이므로 하지 않았다.

**다음으로 고려할 것(미결정, 사용자 판단 대기)**: (a) 진입 로직(C1~C6) 자체를
다른 가설로 다시 설계, (b) 기간을 나눠 장세별(추세장/박스권) 부분 표본을

## GDR 부트스트랩 신뢰구간 (`scripts/run_gdr_bootstrap.py`) — 2026-07-15

## GDR ML 스마트 필터 (`scripts/train_gdr_ml_filter.py`) — 2026-07-15

규칙 기반 가설 4개가 전부 실패한 뒤, 완전히 새 신호 로직 대신 이미 만든
GDR(95종목 격자 최선 조합 f_fill=1.0/k_stop=1.0 — ML 이전에 이미 확정된
선택) 신호 위에서 "이 신호가 이길 확률"을 로지스틱 회귀로 추정해 승률
낮은 신호를 거르는 스마트 필터를 시도했다. 사용자가 표본 크기(1,595건)
대비 딥러닝은 과적합 위험이 크다고 판단해 클래식 ML(로지스틱 회귀)로
시작하기로 결정.

**과적합 방지 설계(전부 결과 확인 전 고정)**: 12개 특징(gap%, ATR%, CLV,
SMA20 괴리, 전일수익률, 거래량비, 5/10/20일 수익률, RSI14, 요일, 월) —
전부 D-1까지 정보로만 계산. 하이퍼파라미터(C=1.0, class_weight=balanced)
그리드서치 없이 고정. 학습/평가는 날짜 기준 시간순 80%/20% 분할(2024-06-27
이후가 OOS) — 미래로 과거를 학습하지 않음. 필터 임계값(P(win)≥0.5)도
사전 고정, 평가는 단 한 번만.

**결과**:

| | 신호수 | 승률 | E_trade(OOS) |
|---|---|---|---|
| 필터 없음(원 GDR) | 373 | 51.47% | −0.4260% |
| ML 필터(P(win)≥0.5) | 184 | 49.46% | −0.4797% |

**정직한 결론 — 두 가지**:

1. **ML 필터는 도움이 안 됐다.** 신호를 절반으로 줄였는데도 승률이
   오히려 떨어졌다(51.47%→49.46%)는 것은 이 12개 특징으로는 GDR 신호의
   승패를 유의미하게 구분하지 못한다는 뜻이다 — "학습 구간 승률
   56.46%"라는 IS 성능이 OOS에서 재현되지 않은, 전형적인 과적합
   징후에 가깝다.
2. **더 중요한 발견**: 필터와 무관하게, OOS 구간(2024-06-27~2026-07)의
   원 GDR E_trade(−0.426%)가 10년 전체 평균(−0.160%)보다 뚜렷이
   나쁘다. 이는 GDR의(이미 작았던) 엣지가 최근 구간에서 더 나빠졌다는
   뜻 — AlgoLab 블로그가 경고했던 "시장 체제 변화(regime change)"가
   실제로 이 유니버스·이 기간에 나타나고 있을 가능성을 시사한다.

이로써 규칙 기반 4개 + ML 필터 1개, 총 5개 접근법이 전부 이 틀(KOSPI
대형주·당일청산·일봉 기반 신호) 안에서 실패했다. 데이터마이닝 없이
정직하게 도달한 현재 결론이다.

## 메타 감사: 데이터 스누핑/다중검정 관점에서 이 프로젝트 재점검 — 2026-07-15

AlgoLab 블로그에 2026-07-15자로 올라온 글("백테스트가 화려할수록 왜
의심해야 할까" — data-snooping/multiple-testing 함정)을 읽고, 지금까지
이 저장소에서 실제로 한 작업을 그 틀로 재점검했다. 핵심 주장: "얼마나
좋은가"가 아니라 "몇 번 시도했는가"를 알아야 결과를 해석할 수 있다 —
100개를 시험하면 실력 없이도 약 5개는 우연히 좋아 보인다.

**실제 시도 횟수 집계**: VCB-Gap(6조합) + GDR(6) + GDR-V(12) + VBP(6) +
ML 필터(1모델) = 파라미터 조합 기준 31개, 가설 계열 기준 5개. 블로그의
경험칙("100개 시험 → 약 5개 가짜 승자")을 대입하면, 31개 규모에서 우연히
좋아 보이는 결과가 1~2개 나오는 건 통계적으로 자연스러운 일이다 —
GDR-V의 "용량-반응 패턴"과 VBP의 "양수 칸" 두 건이 정확히 이 정도
규모다. 유니버스를 20→95종목으로 확장해 재현 여부를 확인한 것이 바로
이 함정에 대한 방어였고, 실제로 둘 다 재현에 실패해 노이즈로 판명났다
— 사후에 보니 이 블로그가 설명하는 메커니즘과 정확히 일치한다.

**그러나 이번 점검으로 드러난 실제 갭**: 블로그의 6가지 방어 규율 중
1번("탐색용 인샘플과 한 번도 안 본 아웃오브샘플 분리")을 우리는 ML
필터(80/20 시간순 분할)에만 적용했다. VCB-Gap·GDR·GDR-V·VBP 네 규칙
기반 전략은 전부 **10년 전체 데이터로 판정**했다 — 처음부터 떼어둔
진짜 홀드아웃이 없었다. 즉 같은 2016~2026 KOSPI 데이터셋에 다섯 개
가설 계열을 순차적으로 물어본 것 자체가, 개별 실험 내부의 규율(사전
등록·전수보고)과 별개로 메타 수준의 다중검정 노출이라는 뜻이다. 이건
소급 적용이 불가능하다 — 이미 이 데이터셋을 다섯 번 들여다봤으므로,
지금부터 홀드아웃을 새로 지정해도 "한 번도 안 본" 구간이 아니다.

**정직한 결론**: 이 10년치 데이터셋 위에서 여섯 번째 가설을 또 시도하는
행위는 근거가 점점 약해진다 — 개별 실험은 깨끗해도 데이터셋 자체가
점점 "탐색용으로 소진"되고 있기 때문이다. 이 데이터셋에서 전략을 계속
찾는 것보다, (a) 앞으로 실제로 쌓일 새 데이터(라이브/페이퍼 트레이딩
기간)를 진짜 한 번도 안 본 홀드아웃으로 남겨두고 시작하거나, (b) 전략
탐색 자체를 여기서 접고 인프라(옵션 3)로 넘어가는 쪽이 이 블로그의
방어 규율과 부합한다. 데이터마이닝 방지 원칙이 그동안 개별 실험
안에서는 잘 지켜졌지만, 이번 점검으로 그 원칙을 프로젝트 전체 수준
으로도 적용해야 한다는 게 분명해졌다.

## 미국 유니버스 확장(20→112종목) 재검증 — 2026-07-15

VCB-Gap의 미국 반전이 KOSPI 때 GDR-V/VBP에서 봤던 "작은 표본의 유망한
패턴"과 같은 함정일 수 있다고 보고, 동일한 원칙(새 조건 추가 아니라
유니버스 확대 후 재검증)으로 S&P500 대형주 92종목을 `EXPANDED_CANDIDATES`

**결과**: VCB-Gap이 표본 기준(1,000/500)을 넘어(2,061신호·1,016거래일)
**처음으로 확정 판정을 받았는데 — 방향이 완전히 뒤집혔다.** 6개 조합
전부 −0.13%~−0.16%로 **reject**. 20종목에서 봤던 전 조합 양수(+0.0004~
+0.1059%)는 재현에 실패했다 — GDR-V의 용량-반응 패턴, VBP의 양수 칸에
이어 **세 번째로 확인된 "작은 표본 유망 패턴 → 큰 표본에서 소멸"** 사례다.

GDR(3,855신호·1,218거래일)도 표본기준을 넘어 확정 reject(−0.29~−0.35%,
KOSPI와 방향 일치). GDR-V는 v_mult=1.2 조합들만 표본기준 통과해 reject,
1.5 조합은 여전히 표본부족. VBP는 아직 표본부족(296신호).

**최종 결론**: KOSPI·미국 두 독립 시장에서 네 가설 전부 음수로 수렴했다.
가설 사냥을 계속하기보다, 표본이 작을 때 나온 그럴듯한 신호는 일단
의심하고 유니버스를 늘려 재검증한다는 이 프로젝트의 원칙이 세 번 연속
정확히 예측대로 작동했다는 것 자체가 지금까지의 가장 중요한 결과다.

## 미국주식 분봉 — KIS와 다른 유리한 조건 (`scripts/collect_minute_bars_us.py`) — 2026-07-15

"미국주식은?"이라는 질문에서 출발 — KIS(KR 섹터)는 분봉을 "오늘 것만"

60일보다 오래된 분은 안 주므로, 롤링 윈도우가 지나가기 전에 주기적으로
재실행해 우리 저장소에 눌러 담아야 과거분이 유실되지 않는다.

수집 스크립트 하나뿐이다. `append_bars`가 (날짜,시각) 기준 중복을
자동으로 걸러내므로, 매번 60일 전체를 재수집해도 안전(멱등)하다.

한정, 7일 후 만료).

**정리**: 한국은 "오늘 것만 매일 쌓기"(느림, 수개월 필요), 미국은
"60일 즉시 백필 + 롤링 유지"(빠름) — 같은 저장소·같은 시장-무관 코드
위에서 시장마다 실제로 다른 데이터 공급 조건에 맞춰 수집 전략만 다르게

미국 분봉을 KIS로도 받을 수 있을지 실계좌로 직접 확인했다. **해외주식
분봉 조회(`HHDFS76950200`, `/uapi/overseas-price/v1/quotations/inquire-
time-itemchartprice`)는 실재하고, 국내 분봉과 달리 하루 안 넘게 여러
날짜에 걸친 데이터를 준다**(5분봉 120개 요청 시 전날 13:50~오늘 07:45가
한 번에 왔음) — 페이지네이션으로 더 과거까지 갈 가능성이 있으나, 짧은
간격으로 반복 호출하다 KIS 토큰 발급 제한(`EGW00133`, 분당 1회)에
걸려 이번엔 깊이 확인은 못 했다.

**algolab.co.kr/platforms/kis**에서 확인한 해외주식 실제 제약 3가지:
- **시장가 주문 불가** — 해외주식은 지정가만 가능
- **Rate Limit 초당 1회** — 국내(초당 20회)의 1/20 수준
- **실시간 시세 WebSocket 41종목 한도**

**키움 비교(algolab.co.kr/platforms/kiwoom + 자체 재조사)**: 처음엔
"키움은 해외주식 API가 아예 없다"고 잘못 말했었다 — 이건 낡은 정보였다.
정정: 키움에는 해외 관련 API가 두 가지다.
1. **해외파생 Open API**(기존) — 해외선물/옵션 전용, 해외주식 미포함,
   Windows 전용(C/C++, C#, VB, Excel, Delphi), WKOA Studio.
2. **해외주식 API**(신규) — 서울경제 보도에 따르면 **2026-07-02**(이
   시점 기준 불과 2주 전) 오픈API 이용약관 개정으로 새로 시작. "AI봇이
   밤새 미국주식도 매매"한다는 보도까지 나왔으나, rate limit·실시간
   한도·지원 거래소 등 **기술 세부사항이 아직 문서화돼 있지 않다** —
   출시 2주짜리 미검증 서비스라 지금 KIS 대신 진지하게 고려하기엔 이르다.

**정리표**:

| | 키움(신규 해외주식 API) | KIS 해외주식 |
|---|---|---|
| 출시/성숙도 | 2026-07-02, 극초기 | 기존 서비스 |
| 기술 세부사항 | 미문서화 | Rate 1회/초, WS 41종목, 지정가만 |
| 국내 rate limit | 권장 초당 5회 | 초당 20회 |
| 플랫폼 | Windows 전용, 32bit(기존 OpenAPI+) / REST는 크로스플랫폼 | 크로스플랫폼, 64bit |

**다음에 생각해볼 아이디어(아직 실행 안 함)**: 사용자가 제안한 "데이터는
키움, 매매는 KIS"처럼 브로커를 역할별로 분리하는 하이브리드 구성 —
AlgoLab 블로그가 언급했던 "다른 증권사 실시간 시세 + 별도 브로커 주문
체결" 패턴과 같은 방향. 아직 비교·검토 단계이며 구현 여부는 결정 안 됨.
